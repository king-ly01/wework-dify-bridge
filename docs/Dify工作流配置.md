# Dify 工作流配置说明

## 工作流要求

### 1. 触发方式
- 使用**「开始」节点**（不是 Webhook 触发节点）

### 2. 开始节点配置
添加输入变量：
- 变量名：`query`
- 类型：`String`
- 描述：用户输入的消息

可选变量：
- 变量名：`user_id`
- 类型：`String`
- 描述：用户ID（用于区分不同用户）

### 3. 工作流编排
在「开始」节点后连接你的处理逻辑，例如：
- LLM 节点
- 知识检索节点
- 工具调用节点
- 等等...

### 4. 结束节点配置
添加输出变量：
- 变量名：`text`
- 类型：`String`
- 值：工作流的最终回复内容

## 示例工作流

```
[开始节点] --query--> [LLM节点] --output--> [结束节点]
                                        text="{{#llm.output#}}"
```

## 测试方法

1. 在 Dify 中点击「运行」
2. 输入测试参数：
   ```json
   {
     "query": "你好"
   }
   ```
3. 确认输出包含 `text` 字段

## 注意事项

1. 工作流必须使用**阻塞模式**（同步执行）
2. 输出变量名必须是 `text`
3. 执行时间不要超过 60 秒（企微超时限制）
